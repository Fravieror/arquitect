What is an array in C# and how does it differ from Python lists or JS arrays?	A C# array is a fixed-size, strongly-typed, contiguous block of memory. Once created, its size cannot change. int[] nums = new int[5]; All elements must be the same type. Contrast with Python lists (dynamic, mixed types) and JS arrays (dynamic, prototype-based). C# arrays are faster for indexed access because memory is contiguous — great for performance-critical code. For dynamic sizing, use List<T> instead.

How do you create and initialize an array in C#?	// Fixed size, default values (0 for int)\nint[] nums = new int[5];\n// Inline initialization\nint[] nums = new int[] { 1, 2, 3, 4, 5 };\n// Shorter syntax (compiler infers size)\nint[] nums = { 1, 2, 3, 4, 5 };\n// 2D array\nint[,] matrix = new int[3, 3];\nSenior insight: arrays are value-type containers but are reference types themselves — assigning an array variable copies the reference, not the data. int[] b = a; means b and a point to the same array.

How do you iterate over an array in C#?	// Classic for loop — use when you need the index\nfor (int i = 0; i < nums.Length; i++)\n    Console.WriteLine(nums[i]);\n// foreach — cleaner, preferred when index not needed\nforeach (int n in nums)\n    Console.WriteLine(n);\n// LINQ — functional style\nnums.ToList().ForEach(Console.WriteLine);\nSenior insight: foreach compiles to an enumerator call (GetEnumerator/MoveNext). For arrays it's optimized by the JIT to be as fast as a for loop. Prefer foreach for readability unless you need the index.

What is List<T> in C# and when should you use it over an array?	List<T> is a generic, dynamically-sized list backed by an array internally. It doubles its capacity when full (amortized O(1) add). Use List<T> when: size is unknown upfront, you need Add/Remove operations. Use array when: size is fixed, maximum performance needed, exposing memory-safe buffers.\nList<int> nums = new List<int>();\nnums.Add(1); nums.Add(2);\nList<int> nums = new List<int> { 1, 2, 3 };\nEquivalent to Python list, JS Array, Java ArrayList.

How do you iterate a List<T> in C#?	List<string> fruits = new List<string> { "apple", "banana", "cherry" };\n// foreach — most common\nforeach (string fruit in fruits)\n    Console.WriteLine(fruit);\n// for with index\nfor (int i = 0; i < fruits.Count; i++)\n    Console.WriteLine($"{i}: {fruits[i]}");\n// LINQ\nfruits.ForEach(f => Console.WriteLine(f));\nNote: use .Count for List<T> and .Length for arrays — mixing them up is a very common mistake for beginners.

How do you use LINQ to transform and filter a list in C#?	LINQ (Language Integrated Query) is C#'s equivalent of Python list comprehensions or JS map/filter:\nvar nums = new List<int> { 1, 2, 3, 4, 5, 6 };\n// Filter\nvar evens = nums.Where(n => n % 2 == 0).ToList();\n// Transform\nvar squares = nums.Select(n => n * n).ToList();\n// Chain\nvar result = nums.Where(n => n > 2).Select(n => n * 10).ToList();\nSenior insight: LINQ is lazy — it doesn't execute until you call .ToList(), .FirstOrDefault(), etc. Calling it multiple times re-executes the query. Materialize with .ToList() when needed more than once.

What is HashSet<T> in C# and what problem does it solve?	HashSet<T> is an unordered collection of unique values backed by a hash table — O(1) average for add, remove, and contains. Think of it as a dictionary with only keys. It guarantees uniqueness automatically. Use it for: fast membership checks, removing duplicates, set math.\nHashSet<string> visited = new HashSet<string>();\nbool isNew = visited.Add("page1"); // returns false if already exists\nbool exists = visited.Contains("page1"); // O(1)\nEquivalent to Python set, Java HashSet, JS Set.

How do you create and populate a HashSet<T> in C#?	// Empty\nvar set = new HashSet<int>();\nset.Add(1); set.Add(2); set.Add(2); // duplicate ignored\n// From collection\nvar list = new List<int> { 1, 2, 2, 3 };\nvar set = new HashSet<int>(list); // { 1, 2, 3 }\n// Inline initialization\nvar set = new HashSet<string> { "a", "b", "c" };\n// Remove\nset.Remove("a");\nSenior insight: Add() returns bool — true if added, false if duplicate. This is useful for deduplication logic without a separate Contains() check.

How do you iterate over a HashSet<T> in C#?	foreach (var item in mySet)\n    Console.WriteLine(item);\nHashSet<T> is unordered — never rely on iteration order. If you need order, sort explicitly:\nforeach (var item in mySet.OrderBy(x => x))\n    Console.WriteLine(item);\nSenior insight: same as Python sets — iteration order is an implementation detail and can change. If order matters, use SortedSet<T> instead (backed by a red-black tree, O(log n) operations).

What set operations does HashSet<T> support in C#?	var a = new HashSet<int> { 1, 2, 3 };\nvar b = new HashSet<int> { 2, 3, 4 };\n// Union — all elements from both\na.UnionWith(b);           // a → { 1, 2, 3, 4 }\n// Intersection — only in both\na.IntersectWith(b);       // a → { 2, 3 }\n// Difference — in a but not b\na.ExceptWith(b);          // a → { 1 }\n// Symmetric difference\na.SymmetricExceptWith(b); // a → { 1, 4 }\nNote: these methods modify the set in-place. Use LINQ set methods if you want non-destructive results.

What is Dictionary<TKey, TValue> in C# and how does it compare to Python dict and JS Map?	Dictionary<TKey, TValue> is a generic hash map with O(1) average get/set/remove. Keys must be unique and implement GetHashCode() and Equals(). Values can be anything.\nDictionary<string, int> scores = new Dictionary<string, int>();\nCompared to: Python dict (same concept, dynamic typing), JS Map (Map is ordered, JS object is unordered), Java HashMap (same concept).\nSenior insight: in C# the key's GetHashCode() and Equals() methods are critical — if you use a custom class as a key, you must override both or you'll get bugs.

How do you create and populate a Dictionary<TKey, TValue> in C#?	// Empty\nvar dict = new Dictionary<string, int>();\ndict["alice"] = 90;\ndict.Add("bob", 85); // throws if key exists\n// Inline initialization\nvar dict = new Dictionary<string, int>\n{\n    { "alice", 90 },\n    { "bob", 85 }\n};\n// Check before add\nif (!dict.ContainsKey("carol"))\n    dict["carol"] = 78;\n// Safe add (no throw)\ndict.TryAdd("carol", 78);

How do you safely access a value in a Dictionary in C#?	// Direct — throws KeyNotFoundException if missing\nint score = dict["alice"];\n// Safe — returns false if key missing\nif (dict.TryGetValue("alice", out int score))\n    Console.WriteLine(score);\n// With default (C# 8+)\nvar score = dict.GetValueOrDefault("alice", 0);\nSenior insight: always use TryGetValue in production code — KeyNotFoundException is a common runtime crash. Think of TryGetValue as the C# equivalent of Python's dict.get("key", default). Direct access is fine only when you're certain the key exists (e.g., after ContainsKey check).

How do you iterate over a Dictionary<TKey, TValue> in C#?	var dict = new Dictionary<string, int> { {"a", 1}, {"b", 2} };\n// Keys and values — most common\nforeach (var kvp in dict)\n    Console.WriteLine($"{kvp.Key}: {kvp.Value}");\n// Deconstruct (C# 7+)\nforeach (var (key, value) in dict)\n    Console.WriteLine($"{key}: {value}");\n// Keys only\nforeach (var key in dict.Keys) ...\n// Values only\nforeach (var value in dict.Values) ...\nSenior insight: since C# 3.5, Dictionary preserves insertion order in practice, but this is NOT guaranteed by the spec. Use SortedDictionary<K,V> if you need sorted keys.

What is the difference between Dictionary, SortedDictionary, and ConcurrentDictionary in C#?	Dictionary<K,V>: O(1) average, unordered, not thread-safe. Use for most cases.\nSortedDictionary<K,V>: O(log n), keys always sorted (red-black tree). Use when sorted iteration matters.\nConcurrentDictionary<K,V>: thread-safe, lock-free reads. Use in multi-threaded scenarios.\n// Thread-safe upsert\nvar cd = new ConcurrentDictionary<string, int>();\ncd.AddOrUpdate("key", 1, (k, old) => old + 1);\nSenior insight: never share a plain Dictionary across threads without locking — it can corrupt internally. ConcurrentDictionary has fine-grained locking and is far safer.

Does C# .NET have a concept equivalent to Python virtual environments?	Not exactly — .NET handles dependency isolation differently. Each project has its own .csproj file that specifies its dependencies (NuGet packages). When you build, packages are restored per-project. The equivalent concepts are:\n1. NuGet packages — managed per project via .csproj\n2. SDK version — controlled via global.json\n3. Docker containers — strongest isolation (separate runtime)\nUnlike Python, there's no shared global package installation problem by default — .NET projects are self-contained. global.json pins the SDK version across a repo.

How do you manage package dependencies in a .NET project?	// Add a package\ndotnet add package Newtonsoft.Json\ndotnet add package Microsoft.EntityFrameworkCore --version 8.0.0\n// Restore packages (like pip install -r requirements.txt)\ndotnet restore\n// List packages\ndotnet list package\n// Update\ndotnet add package PackageName --version x.x.x\nPackages are defined in .csproj:\n<PackageReference Include="Newtonsoft.Json" Version="13.0.3" />\nSenior insight: pin exact versions in production. Use NuGet.lock file (dotnet restore --use-lock-file) for reproducible builds — equivalent to package-lock.json in Node.

How do you control which .NET SDK version a project uses (like pyenv for Python)?	Create a global.json file in the solution root:\n{\n  "sdk": {\n    "version": "8.0.100",\n    "rollForward": "latestPatch"\n  }\n}\ndotnet --version  // confirms which SDK is active\nrollForward options: disable (exact version only), latestPatch (safe), latestMinor, latestMajor.\nSenior insight: always add global.json in team projects. Without it, each developer might use a different SDK — leading to subtle build differences. This is the closest C# equivalent to .python-version or pyenv.

What is ASP.NET Core and what makes it different from ASP.NET Framework?	ASP.NET Core is a cross-platform, high-performance, open-source web framework. Key differences from old ASP.NET Framework:\n1. Cross-platform: runs on Linux, Mac, Windows (old was Windows-only)\n2. Modular: no System.Web monolith — add only what you need\n3. Built-in DI: first-class dependency injection container\n4. High performance: consistently top-ranked in TechEmpower benchmarks\n5. Unified: MVC + Web API + Razor Pages + SignalR in one framework\nSenior insight: for new projects, always use ASP.NET Core. .NET Framework is legacy.

How do you create a minimal REST API in ASP.NET Core (.NET 6+)?	// Program.cs — Minimal API style\nvar builder = WebApplication.CreateBuilder(args);\nvar app = builder.Build();\n\napp.MapGet("/", () => "Hello World!");\napp.MapGet("/items/{id}", (int id) => new { Id = id, Name = "Widget" });\napp.MapPost("/items", (Item item) => Results.Created($"/items/1", item));\n\napp.Run();\nThis is the minimal hosting model introduced in .NET 6 — no Startup.cs needed. Equivalent to FastAPI's @app.get() decorators or Express's app.get() in Node.

How do you create a controller-based REST API in ASP.NET Core?	[ApiController]\n[Route("api/[controller]")]\npublic class UsersController : ControllerBase\n{\n    [HttpGet]\n    public IActionResult GetAll() => Ok(new[] { "Alice", "Bob" });\n\n    [HttpGet("{id:int}")]\n    public IActionResult GetById(int id) => Ok(new { Id = id });\n\n    [HttpPost]\n    public IActionResult Create([FromBody] UserDto dto)\n        => CreatedAtAction(nameof(GetById), new { id = 1 }, dto);\n}\nSenior insight: [ApiController] enables automatic model validation, automatic 400 responses for invalid input, and binding source inference — always use it for APIs.

What is the difference between Minimal APIs and Controller-based APIs in ASP.NET Core?	Minimal APIs (net6+): less boilerplate, great for microservices and simple endpoints. Performance slightly better due to less middleware overhead.\nController APIs: more structure, better for large teams, easier to apply cross-cutting concerns (filters, auth policies) per controller.\nChoose Minimal APIs when: small service, few endpoints, prototyping.\nChoose Controllers when: complex API, team convention, need filters/action results.\nSenior insight: both can coexist in the same app. Many teams use minimal APIs for internal routes and controllers for public-facing API surface.

How do you set up routing, model binding, and validation in ASP.NET Core?	// Route templates\n[HttpGet("users/{id:int}")]   // path param, constrained to int\n[HttpGet("search")]           // query param: /search?q=hello\n// Model binding sources\npublic IActionResult Create(\n    [FromRoute] int id,\n    [FromQuery] string filter,\n    [FromBody] CreateUserDto dto,\n    [FromHeader(Name = "X-Api-Key")] string apiKey)\n// Validation via DataAnnotations\npublic class CreateUserDto {\n    [Required] public string Name { get; set; }\n    [Range(18, 120)] public int Age { get; set; }\n    [EmailAddress] public string Email { get; set; }\n}\n[ApiController] auto-validates and returns 400 on failure.

How does Dependency Injection work in ASP.NET Core?	DI is built into ASP.NET Core — no third-party container needed. Register services in Program.cs:\nbuilder.Services.AddScoped<IUserRepository, UserRepository>();\nbuilder.Services.AddSingleton<IEmailService, SmtpEmailService>();\nbuilder.Services.AddTransient<IValidator, UserValidator>();\nThen inject via constructor:\npublic class UsersController(IUserRepository repo) : ControllerBase { ... }\nLifetimes:\n- Singleton: one instance for app lifetime\n- Scoped: one per HTTP request (default for DB contexts)\n- Transient: new instance every time injected\nSenior insight: never inject Transient or Scoped into Singleton — it causes captive dependency bugs.

How do you add middleware in ASP.NET Core and what is the middleware pipeline?	Middleware is code that runs on every request/response in a pipeline — like a chain of responsibility pattern. Order matters.\nvar app = builder.Build();\napp.UseHttpsRedirection();  // redirect HTTP to HTTPS\napp.UseAuthentication();    // who are you?\napp.UseAuthorization();     // what can you do?\napp.UseMiddleware<RequestLoggingMiddleware>();\napp.MapControllers();\nCustom middleware:\napp.Use(async (context, next) => {\n    // before\n    await next(context);\n    // after\n});\nSenior insight: UseAuthentication must come before UseAuthorization — swapping them breaks auth silently.

How do you implement async endpoints in ASP.NET Core?	[HttpGet("{id}")]\npublic async Task<IActionResult> GetUser(int id)\n{\n    var user = await _repository.GetByIdAsync(id);\n    if (user is null) return NotFound();\n    return Ok(user);\n}\nAlways use async/await for I/O operations (database, HTTP calls, file reads). ASP.NET Core uses a thread pool — async frees threads during I/O, allowing the server to handle more concurrent requests with fewer threads.\nSenior insight: avoid async void (except event handlers). Always return Task or Task<T>. Mixing sync and async (calling .Result or .Wait()) can cause deadlocks in ASP.NET Core — always await.

What is gRPC in .NET and how do you set it up?	gRPC in .NET uses the Grpc.AspNetCore package and integrates directly into the ASP.NET Core host. It uses Protocol Buffers for serialization over HTTP/2. Benefits over REST: binary (smaller, faster), strongly typed contracts, streaming support.\n// Install packages\ndotnet add package Grpc.AspNetCore\n// In Program.cs\nbuilder.Services.AddGrpc();\napp.MapGrpcService<GreeterService>();\nServer requires HTTP/2 — configure Kestrel accordingly or use TLS (which enables HTTP/2 by default).

How do you define a gRPC service contract in a .proto file for .NET?	Create greeter.proto in the Protos/ folder:\nsyntax = "proto3";\noption csharp_namespace = "MyApp.Grpc";\npackage greeter;\n\nservice Greeter {\n  rpc SayHello (HelloRequest) returns (HelloReply);\n}\nmessage HelloRequest { string name = 1; }\nmessage HelloReply { string message = 1; }\nAdd to .csproj:\n<ItemGroup>\n  <Protobuf Include="Protos\\greeter.proto" GrpcServices="Both" />\n</ItemGroup>\nGrpcServices="Both" generates both server and client code. Use "Server" or "Client" to generate only what's needed.

How do you implement a gRPC server in ASP.NET Core?	// Generated base class from proto\nusing Grpc.Core;\nusing MyApp.Grpc;\n\npublic class GreeterService : Greeter.GreeterBase\n{\n    public override Task<HelloReply> SayHello(\n        HelloRequest request,\n        ServerCallContext context)\n    {\n        return Task.FromResult(new HelloReply\n        {\n            Message = $"Hello, {request.Name}!"\n        });\n    }\n}\n// Register in Program.cs\nbuilder.Services.AddGrpc();\napp.MapGrpcService<GreeterService>();\nSenior insight: ServerCallContext gives access to metadata, deadlines, and cancellation tokens — always check context.CancellationToken for long-running operations.

How do you create a gRPC client in .NET?	// Factory pattern — recommended\nbuilder.Services.AddGrpcClient<Greeter.GreeterClient>(o =>\n{\n    o.Address = new Uri("https://localhost:7042");\n});\n// Inject and use\npublic class MyService(Greeter.GreeterClient client)\n{\n    public async Task CallGreeter()\n    {\n        var reply = await client.SayHelloAsync(\n            new HelloRequest { Name = "Alice" });\n        Console.WriteLine(reply.Message);\n    }\n}\nSenior insight: always use AddGrpcClient() (the factory pattern) instead of creating channels manually — it integrates with DI, handles channel reuse, and supports HttpClientFactory retry/logging policies.

How do you handle errors in gRPC .NET?	gRPC uses RpcException with StatusCode instead of HTTP status codes:\n// Server — throw RpcException\npublic override Task<UserReply> GetUser(UserRequest req, ServerCallContext ctx)\n{\n    if (user is null)\n        throw new RpcException(new Status(StatusCode.NotFound, "User not found"));\n    return Task.FromResult(new UserReply { Name = user.Name });\n}\n// Client — catch RpcException\ntry {\n    var reply = await client.GetUserAsync(new UserRequest { Id = 1 });\n} catch (RpcException ex) when (ex.StatusCode == StatusCode.NotFound) {\n    Console.WriteLine("Not found: " + ex.Status.Detail);\n}\nCommon codes: NotFound, InvalidArgument, Unauthenticated, PermissionDenied, Internal, Unavailable.

What are the 4 gRPC communication patterns and how are they defined in .proto?	1. Unary — one request, one response:\n   rpc GetUser (Request) returns (Response);\n2. Server streaming — one request, stream of responses:\n   rpc ListUsers (Request) returns (stream Response);\n3. Client streaming — stream of requests, one response:\n   rpc UploadData (stream Request) returns (Response);\n4. Bidirectional streaming — both sides stream:\n   rpc Chat (stream Message) returns (stream Message);\nIn C# server streaming:\npublic override async Task ListUsers(Request req, IServerStreamWriter<Response> stream, ServerCallContext ctx) {\n    foreach (var user in users)\n        await stream.WriteAsync(new Response { Name = user.Name });\n}

How do you configure Kestrel to support gRPC and HTTP/2 in .NET?	gRPC requires HTTP/2. In production with TLS, this is automatic. For development without TLS:\n// Program.cs\nbuilder.WebHost.ConfigureKestrel(options =>\n{\n    options.ListenLocalhost(5001, o => o.Protocols = HttpProtocols.Http2);\n    options.ListenLocalhost(5000, o => o.Protocols = HttpProtocols.Http1); // for REST\n});\nOr in appsettings.json under Kestrel.Endpoints. Senior insight: you can host gRPC and REST on different ports in the same process. HTTP/1.1 and HTTP/2 cannot share the same port without TLS ALPN negotiation.

What is the difference between record, class, and struct in C# for data models?	class: reference type, heap-allocated, nullable, mutable by default. Use for complex objects with behavior.\nrecord (C# 9+): reference type, immutable by default, value-based equality, with-expression for copying. Best for DTOs and immutable data.\nstruct: value type, stack-allocated, copied on assignment. Use for small, short-lived data (coordinates, RGB values).\n// Record\npublic record UserDto(string Name, int Age);\nvar u2 = u1 with { Age = 31 }; // non-destructive mutation\nSenior insight: use records for API request/response DTOs — they're immutable, equality just works, and they deconstruct nicely.

What are common C# mistakes when working with collections?	1. Modifying a list while iterating:\nforeach (var item in list) list.Remove(item); // InvalidOperationException\n// Fix: list.RemoveAll(x => condition);\n2. Using List<T> for frequent Contains() checks:\nlist.Contains(x) // O(n) — use HashSet<T> for O(1)\n3. Confusing Count (List) with Length (array):\nnums.Length  // array\nnums.Count   // List<T>\n4. Not disposing IEnumerable from LINQ:\nvar q = db.Users.Where(x => x.Active); // not executed yet\nforeach (var u in q) ... // executes SQL here\n// Calling q again re-executes the query!

What is the difference between IEnumerable, ICollection, IList, and List<T>?	IEnumerable<T>: read-only, forward-only iteration. No Count, no index access. Laziest — use as return type for flexibility.\nICollection<T>: adds Count, Add, Remove, Contains.\nIList<T>: adds index access (list[i]) and Insert.\nList<T>: concrete implementation of IList<T> with extra methods (Sort, Find, etc.).\nSenior insight: return IEnumerable<T> from methods when callers only need to iterate — it lets you swap implementations (array, list, generator) without breaking callers. Accept the most specific type you need as parameters. Liskov substitution principle in practice.

How does async/await work in C# and what is the difference from Python and JS?	C# async/await uses Task-based Asynchronous Pattern (TAP). An async method returns Task or Task<T> — a promise of future work.\nasync Task<string> FetchDataAsync() {\n    var result = await httpClient.GetStringAsync(url);\n    return result;\n}\nComparisons:\n- JS: single-threaded event loop, async is the only concurrency model\n- Python asyncio: cooperative, single-threaded event loop\n- C#: uses thread pool, true parallelism possible with Task.WhenAll\nSenior insight: in C#, ConfigureAwait(false) in library code avoids deadlocks by not capturing the synchronization context — always use it in non-UI library methods.

What is the difference between Task.WhenAll and Task.WhenAny in C#?	// WhenAll — wait for ALL tasks to complete (parallel)\nvar tasks = users.Select(u => SendEmailAsync(u));\nawait Task.WhenAll(tasks); // all run concurrently\n// WhenAny — complete when FIRST task finishes\nvar winner = await Task.WhenAny(task1, task2, task3);\nWhenAll is like Promise.all() in JS or asyncio.gather() in Python.\nWhenAny is like Promise.race() in JS.\nSenior insight: WhenAll is the key to parallel I/O in C# — running 10 DB queries sequentially vs concurrently can be the difference between 1000ms and 100ms response time.

What are common REST API best practices in ASP.NET Core?	1. Use [ProducesResponseType] to document all possible response codes\n2. Return IActionResult or ActionResult<T> — not raw objects\n3. Use ProblemDetails (RFC 7807) for error responses — built into ASP.NET Core\n4. Implement versioning from day one: /api/v1/users\n5. Use DTOs — never expose your domain/EF entities directly\n6. Return 201 Created with Location header for POST\n7. Use CancellationToken in all async endpoints:\npublic async Task<IActionResult> Get(CancellationToken ct)\n    => Ok(await _repo.GetAllAsync(ct));\nSenior insight: CancellationToken lets the server abort work when the client disconnects — critical for expensive DB queries.

How do you add authentication and authorization to an ASP.NET Core REST API?	// JWT Bearer auth setup in Program.cs\nbuilder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)\n    .AddJwtBearer(options => {\n        options.Authority = "https://your-identity-provider";\n        options.Audience = "your-api";\n    });\nbuilder.Services.AddAuthorization();\n// In pipeline (ORDER MATTERS)\napp.UseAuthentication();\napp.UseAuthorization();\n// On endpoints\n[Authorize]                          // any authenticated user\n[Authorize(Roles = "Admin")]         // role-based\n[Authorize(Policy = "RequireEmail")] // policy-based\n[AllowAnonymous]                     // override for public endpoints
