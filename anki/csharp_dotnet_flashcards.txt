What is an array in C# and how does it differ from Python lists or JS arrays?	<div style="text-align:left">A C# array is a fixed-size, strongly-typed, contiguous block of memory. Once created, its size cannot change. <code>int[] nums = new int[5];</code> All elements must be the same type. Contrast with Python lists (dynamic, mixed types) and JS arrays (dynamic, prototype-based). C# arrays are faster for indexed access because memory is contiguous — great for performance-critical code. For dynamic sizing, use <code>List&lt;T&gt;</code> instead.</div>
How do you create and initialize an array in C#?	<div style="text-align:left"><code>// Fixed size, default values (0 for int)<br>int[] nums = new int[5];<br>// Inline initialization<br>int[] nums = new int[] { 1, 2, 3, 4, 5 };<br>// Shorter syntax (compiler infers size)<br>int[] nums = { 1, 2, 3, 4, 5 };<br>// 2D array<br>int[,] matrix = new int[3, 3];</code><br><br>Senior insight: arrays are value-type containers but are reference types themselves — assigning an array variable copies the reference, not the data. <code>int[] b = a;</code> means b and a point to the same array.</div>
How do you iterate over an array in C#?	<div style="text-align:left"><code>// Classic for loop — use when you need the index<br>for (int i = 0; i &lt; nums.Length; i++)<br>    Console.WriteLine(nums[i]);<br>// foreach — cleaner, preferred when index not needed<br>foreach (int n in nums)<br>    Console.WriteLine(n);<br>// LINQ — functional style<br>nums.ToList().ForEach(Console.WriteLine);</code><br><br>Senior insight: <code>foreach</code> compiles to an enumerator call (GetEnumerator/MoveNext). For arrays it's optimized by the JIT to be as fast as a <code>for</code> loop. Prefer <code>foreach</code> for readability unless you need the index.</div>
What is List&lt;T&gt; in C# and when should you use it over an array?	<div style="text-align:left"><code>List&lt;T&gt;</code> is a generic, dynamically-sized list backed by an array internally. It doubles its capacity when full (amortized O(1) add).<br>Use <code>List&lt;T&gt;</code> when: size is unknown upfront, you need <code>.Add()</code>/<code>.Remove()</code> operations.<br>Use array when: size is fixed, maximum performance needed, exposing memory-safe buffers.<br><code>List&lt;int&gt; nums = new List&lt;int&gt;();<br>nums.Add(1); nums.Add(2);<br>List&lt;int&gt; nums = new List&lt;int&gt; { 1, 2, 3 };</code><br>Equivalent to Python list, JS Array, Java ArrayList.</div>
How do you iterate a List&lt;T&gt; in C#?	<div style="text-align:left"><code>List&lt;string&gt; fruits = new List&lt;string&gt; { "apple", "banana", "cherry" };<br>// foreach — most common<br>foreach (string fruit in fruits)<br>    Console.WriteLine(fruit);<br>// for with index<br>for (int i = 0; i &lt; fruits.Count; i++)<br>    Console.WriteLine($"{i}: {fruits[i]}");<br>// LINQ<br>fruits.ForEach(f => Console.WriteLine(f));</code><br><br>Note: use <code>.Count</code> for <code>List&lt;T&gt;</code> and <code>.Length</code> for arrays — mixing them up is a very common mistake for beginners.</div>
How do you use LINQ to transform and filter a list in C#?	<div style="text-align:left">LINQ (Language Integrated Query) is C#'s equivalent of Python list comprehensions or JS map/filter:<br><code>var nums = new List&lt;int&gt; { 1, 2, 3, 4, 5, 6 };<br>// Filter<br>var evens = nums.Where(n => n % 2 == 0).ToList();<br>// Transform<br>var squares = nums.Select(n => n * n).ToList();<br>// Chain<br>var result = nums.Where(n => n &gt; 2).Select(n => n * 10).ToList();</code><br><br>Senior insight: LINQ is lazy — it doesn't execute until you call <code>.ToList()</code>, <code>.FirstOrDefault()</code>, etc. Calling it multiple times re-executes the query. Materialize with <code>.ToList()</code> when needed more than once.</div>
What is HashSet&lt;T&gt; in C# and what problem does it solve?	<div style="text-align:left"><code>HashSet&lt;T&gt;</code> is an unordered collection of unique values backed by a hash table — O(1) average for add, remove, and contains. Think of it as a dictionary with only keys. It guarantees uniqueness automatically. Use it for: fast membership checks, removing duplicates, set math.<br><code>HashSet&lt;string&gt; visited = new HashSet&lt;string&gt;();<br>bool isNew = visited.Add("page1"); // returns false if already exists<br>bool exists = visited.Contains("page1"); // O(1)</code><br>Equivalent to Python set, Java HashSet, JS Set.</div>
How do you create and populate a HashSet&lt;T&gt; in C#?	<div style="text-align:left"><code>// Empty<br>var set = new HashSet&lt;int&gt;();<br>set.Add(1); set.Add(2); set.Add(2); // duplicate ignored<br>// From collection<br>var list = new List&lt;int&gt; { 1, 2, 2, 3 };<br>var set = new HashSet&lt;int&gt;(list); // { 1, 2, 3 }<br>// Inline initialization<br>var set = new HashSet&lt;string&gt; { "a", "b", "c" };<br>// Remove<br>set.Remove("a");</code><br><br>Senior insight: <code>.Add()</code> returns <code>bool</code> — <code>true</code> if added, <code>false</code> if duplicate. This is useful for deduplication logic without a separate <code>.Contains()</code> check.</div>
How do you iterate over a HashSet&lt;T&gt; in C#?	<div style="text-align:left"><code>foreach (var item in mySet)<br>    Console.WriteLine(item);</code><br><code>HashSet&lt;T&gt;</code> is unordered — never rely on iteration order. If you need order, sort explicitly:<br><code>foreach (var item in mySet.OrderBy(x => x))<br>    Console.WriteLine(item);</code><br><br>Senior insight: same as Python sets — iteration order is an implementation detail and can change. If order matters, use <code>SortedSet&lt;T&gt;</code> instead (backed by a red-black tree, O(log n) operations).</div>
What set operations does HashSet&lt;T&gt; support in C#?	<div style="text-align:left"><code>var a = new HashSet&lt;int&gt; { 1, 2, 3 };<br>var b = new HashSet&lt;int&gt; { 2, 3, 4 };<br>// Union — all elements from both<br>a.UnionWith(b);           // a → { 1, 2, 3, 4 }<br>// Intersection — only in both<br>a.IntersectWith(b);       // a → { 2, 3 }<br>// Difference — in a but not b<br>a.ExceptWith(b);          // a → { 1 }<br>// Symmetric difference<br>a.SymmetricExceptWith(b); // a → { 1, 4 }</code><br><br>Note: these methods modify the set in-place. Use LINQ set methods if you want non-destructive results.</div>
What is Dictionary&lt;TKey, TValue&gt; in C# and how does it compare to Python dict and JS Map?	<div style="text-align:left"><code>Dictionary&lt;TKey, TValue&gt;</code> is a generic hash map with O(1) average get/set/remove. Keys must be unique and implement <code>GetHashCode()</code> and <code>Equals()</code>. Values can be anything.<br><code>Dictionary&lt;string, int&gt; scores = new Dictionary&lt;string, int&gt;();</code><br>Compared to: Python dict (same concept, dynamic typing), JS Map (Map is ordered, JS object is unordered), Java HashMap (same concept).<br><br>Senior insight: in C# the key's <code>GetHashCode()</code> and <code>Equals()</code> methods are critical — if you use a custom <code>class</code> as a key, you must override both or you'll get bugs.</div>
How do you create and populate a Dictionary&lt;TKey, TValue&gt; in C#?	<div style="text-align:left"><code>// Empty<br>var dict = new Dictionary&lt;string, int&gt;();<br>dict["alice"] = 90;<br>dict.Add("bob", 85); // throws if key exists<br>// Inline initialization<br>var dict = new Dictionary&lt;string, int&gt;<br>{<br>    { "alice", 90 },<br>    { "bob", 85 }<br>};<br>// Check before add<br>if (!dict.ContainsKey("carol"))<br>    dict["carol"] = 78;<br>// Safe add (no throw)<br>dict.TryAdd("carol", 78);</code></div>
How do you safely access a value in a Dictionary in C#?	<div style="text-align:left"><code>// Direct — throws KeyNotFoundException if missing<br>int score = dict["alice"];<br>// Safe — returns false if key missing<br>if (dict.TryGetValue("alice", out int score))<br>    Console.WriteLine(score);<br>// With default (C# 8+)<br>var score = dict.GetValueOrDefault("alice", 0);</code><br><br>Senior insight: always use <code>.TryGetValue()</code> in production code — <code>KeyNotFoundException</code> is a common runtime crash. Think of <code>.TryGetValue()</code> as the C# equivalent of Python's <code>dict.get("key", default)</code>. Direct access is fine only when you're certain the key exists (e.g., after <code>.ContainsKey()</code> check).</div>
How do you iterate over a Dictionary&lt;TKey, TValue&gt; in C#?	<div style="text-align:left"><code>var dict = new Dictionary&lt;string, int&gt; { {"a", 1}, {"b", 2} };<br>// Keys and values — most common<br>foreach (var kvp in dict)<br>    Console.WriteLine($"{kvp.Key}: {kvp.Value}");<br>// Deconstruct (C# 7+)<br>foreach (var (key, value) in dict)<br>    Console.WriteLine($"{key}: {value}");<br>// Keys only<br>foreach (var key in dict.Keys) ...<br>// Values only<br>foreach (var value in dict.Values) ...</code><br><br>Senior insight: since C# 3.5, Dictionary preserves insertion order in practice, but this is NOT guaranteed by the spec. Use <code>SortedDictionary&lt;K,V&gt;</code> if you need sorted keys.</div>
What is the difference between Dictionary, SortedDictionary, and ConcurrentDictionary in C#?	<div style="text-align:left"><code>Dictionary&lt;K,V&gt;</code>: O(1) average, unordered, not thread-safe. Use for most cases.<br><code>SortedDictionary&lt;K,V&gt;</code>: O(log n), keys always sorted (red-black tree). Use when sorted iteration matters.<br><code>ConcurrentDictionary&lt;K,V&gt;</code>: thread-safe, lock-free reads. Use in multi-threaded scenarios.<br><code>// Thread-safe upsert<br>var cd = new ConcurrentDictionary&lt;string, int&gt;();<br>cd.AddOrUpdate("key", 1, (k, old) => old + 1);</code><br><br>Senior insight: never share a plain Dictionary across threads without locking — it can corrupt internally. <code>ConcurrentDictionary&lt;K,V&gt;</code> has fine-grained locking and is far safer.</div>
Does C# .NET have a concept equivalent to Python virtual environments?	<div style="text-align:left">Not exactly — .NET handles dependency isolation differently. Each project has its own <code>.csproj</code> file that specifies its dependencies (NuGet packages). When you build, packages are restored per-project. The equivalent concepts are:<br>1. NuGet packages — managed per project via <code>.csproj</code><br>2. SDK version — controlled via <code>global.json</code><br>3. Docker containers — strongest isolation (separate runtime)<br>Unlike Python, there's no shared global package installation problem by default — .NET projects are self-contained. <code>global.json</code> pins the SDK version across a repo.</div>
How do you manage package dependencies in a .NET project?	<div style="text-align:left"><code>// Add a package<br>dotnet add package Newtonsoft.Json<br>dotnet add package Microsoft.EntityFrameworkCore --version 8.0.0<br>// Restore packages (like pip install -r requirements.txt)<br>dotnet restore<br>// List packages<br>dotnet list package<br>// Update<br>dotnet add package PackageName --version x.x.x</code><br>Packages are defined in <code>.csproj</code>:<br><code>&lt;PackageReference Include="Newtonsoft.Json" Version="13.0.3" /&gt;</code><br><br>Senior insight: pin exact versions in production. Use <code>NuGet.lock</code> file (<code>dotnet restore</code> --use-lock-file) for reproducible builds — equivalent to <code>package-lock.json</code> in Node.</div>
How do you control which .NET SDK version a project uses (like pyenv for Python)?	<div style="text-align:left">Create a <code>global.json</code> file in the solution root:<br><code>{<br>  "sdk": {<br>    "version": "8.0.100",<br>    "rollForward": "latestPatch"<br>  }<br>}</code><br><code>dotnet --version</code>  // confirms which SDK is active<br><br>rollForward options: <code>disable</code> (exact version only), <code>latestPatch</code> (safe), <code>latestMinor</code>, <code>latestMajor</code>.<br><br>Senior insight: always add <code>global.json</code> in team projects. Without it, each developer might use a different SDK — leading to subtle build differences. This is the closest C# equivalent to <code>.python-version</code> or pyenv.</div>
What is ASP.NET Core and what makes it different from ASP.NET Framework?	<div style="text-align:left">ASP.NET Core is a cross-platform, high-performance, open-source web framework. Key differences from old ASP.NET Framework:<br>1. Cross-platform: runs on Linux, Mac, Windows (old was Windows-only)<br>2. Modular: no System.Web monolith — add only what you need<br>3. Built-in DI: first-class dependency injection container<br>4. High performance: consistently top-ranked in TechEmpower benchmarks<br>5. Unified: MVC + Web API + Razor Pages + SignalR in one framework<br><br>Senior insight: for new projects, always use ASP.NET Core. .NET Framework is legacy.</div>
How do you create a minimal REST API in ASP.NET Core (.NET 6+)?	<div style="text-align:left"><code>// Program.cs — Minimal API style<br>var builder = WebApplication.CreateBuilder(args);<br>var app = builder.Build();<br>app.MapGet("/", () => "Hello World!");<br>app.MapGet("/items/{id}", (int id) => new { Id = id, Name = "Widget" });<br>app.MapPost("/items", (Item item) => Results.Created($"/items/1", item));<br>app.Run();</code><br>This is the minimal hosting model introduced in .NET 6 — no Startup.cs needed. Equivalent to FastAPI's @app.get() decorators or Express's app.get() in Node.</div>
How do you create a controller-based REST API in ASP.NET Core?	<div style="text-align:left"><code>[ApiController]<br>[Route("api/[controller]")]<br>public class UsersController : ControllerBase<br>{<br>    [HttpGet]<br>    public IActionResult GetAll() => Ok(new[] { "Alice", "Bob" });<br>    [HttpGet("{id:int}")]<br>    public IActionResult GetById(int id) => Ok(new { Id = id });<br>    [HttpPost]<br>    public IActionResult Create([FromBody] UserDto dto)<br>        => CreatedAtAction(nameof(GetById), new { id = 1 }, dto);<br>}</code><br><br>Senior insight: <code>[ApiController]</code> enables automatic model validation, automatic 400 responses for invalid input, and binding source inference — always use it for APIs.</div>
What is the difference between Minimal APIs and Controller-based APIs in ASP.NET Core?	<div style="text-align:left">Minimal APIs (.NET 6+): less boilerplate, great for microservices and simple endpoints. Performance slightly better due to less middleware overhead.<br>Controller APIs: more structure, better for large teams, easier to apply cross-cutting concerns (filters, auth policies) per controller.<br><br>Choose Minimal APIs when: small service, few endpoints, prototyping.<br>Choose Controllers when: complex API, team convention, need filters/action results.<br><br>Senior insight: both can coexist in the same app. Many teams use minimal APIs for internal routes and controllers for public-facing API surface.</div>
How do you set up routing, model binding, and validation in ASP.NET Core?	<div style="text-align:left"><code>// Route templates<br>[HttpGet("users/{id:int}")]   // path param, constrained to int<br>[HttpGet("search")]           // query param: /search?q=hello<br>// Model binding sources<br>public IActionResult Create(<br>    [FromRoute] int id,<br>    [FromQuery] string filter,<br>    [FromBody] CreateUserDto dto,<br>    [FromHeader(Name = "X-Api-Key")] string apiKey)<br>// Validation via DataAnnotations<br>public class CreateUserDto {<br>    [Required] public string Name { get; set; }<br>    [Range(18, 120)] public int Age { get; set; }<br>    [EmailAddress] public string Email { get; set; }<br>}</code><br><code>[ApiController]</code> auto-validates and returns 400 on failure.</div>
How does Dependency Injection work in ASP.NET Core?	<div style="text-align:left">DI is built into ASP.NET Core — no third-party container needed. Register services in Program.cs:<br><code>builder.Services.AddScoped&lt;IUserRepository, UserRepository&gt;();<br>builder.Services.AddSingleton&lt;IEmailService, SmtpEmailService&gt;();<br>builder.Services.AddTransient&lt;IValidator, UserValidator&gt;();</code><br>Then inject via constructor:<br><code>public class UsersController(IUserRepository repo) : ControllerBase { ... }</code><br><br>Lifetimes:<br>- <code>AddSingleton</code>: one instance for app lifetime<br>- <code>AddScoped</code>: one per HTTP request (default for DB contexts)<br>- <code>AddTransient</code>: new instance every time injected<br><br>Senior insight: never inject <code>AddTransient</code> or <code>AddScoped</code> into <code>AddSingleton</code> — it causes captive dependency bugs.</div>
How do you add middleware in ASP.NET Core and what is the middleware pipeline?	<div style="text-align:left">Middleware is code that runs on every request/response in a pipeline — like a chain of responsibility pattern. Order matters.<br><code>var app = builder.Build();<br>app.UseHttpsRedirection();  // redirect HTTP to HTTPS<br>app.UseAuthentication();    // who are you?<br>app.UseAuthorization();     // what can you do?<br>app.UseMiddleware&lt;RequestLoggingMiddleware&gt;();<br>app.MapControllers();</code><br>Custom middleware:<br><code>app.Use(async (context, next) => {<br>    // before<br>    await next(context);<br>    // after<br>});</code><br><br>Senior insight: <code>UseAuthentication</code> must come before <code>UseAuthorization</code> — swapping them breaks auth silently.</div>
How do you implement async endpoints in ASP.NET Core?	<div style="text-align:left"><code>[HttpGet("{id}")]<br>public async Task&lt;IActionResult&gt; GetUser(int id)<br>{<br>    var user = await _repository.GetByIdAsync(id);<br>    if (user is null) return NotFound();<br>    return Ok(user);<br>}</code><br>Always use <code>async</code>/<code>await</code> for I/O operations (database, HTTP calls, file reads). ASP.NET Core uses a thread pool — <code>async</code> frees threads during I/O, allowing the server to handle more concurrent requests with fewer threads.<br><br>Senior insight: avoid <code>async</code> <code>void</code> (except event handlers). Always return <code>Task</code> or <code>Task&lt;T&gt;</code>. Mixing sync and async (calling <code>.Result</code> or <code>.Wait()</code>) can cause deadlocks in ASP.NET Core — always <code>await</code>.</div>
What is gRPC in .NET and how do you set it up?	<div style="text-align:left">gRPC in .NET uses the Grpc.AspNetCore package and integrates directly into the ASP.NET Core host. It uses Protocol Buffers for serialization over HTTP/2. Benefits over REST: binary (smaller, faster), strongly typed contracts, streaming support.<br><code>// Install packages<br>dotnet add package Grpc.AspNetCore<br>// In Program.cs<br>builder.Services.AddGrpc();<br>app.MapGrpcService&lt;GreeterService&gt;();</code><br>Server requires HTTP/2 — configure Kestrel accordingly or use TLS (which enables HTTP/2 by default).</div>
How do you define a gRPC service contract in a .proto file for .NET?	<div style="text-align:left">Create greeter.proto in the Protos/ folder:<br><code>syntax = "proto3";<br>option csharp_namespace = "MyApp.Grpc";<br>package greeter;<br>service Greeter {<br>  rpc SayHello (HelloRequest) returns (HelloReply);<br>}<br>message HelloRequest { string name = 1; }<br>message HelloReply { string message = 1; }</code><br>Add to <code>.csproj</code>:<br><code>&lt;ItemGroup&gt;<br>  &lt;Protobuf Include="Protos\\greeter.proto" GrpcServices="Both" /&gt;<br>&lt;/ItemGroup&gt;</code><br><code>GrpcServices="Both"</code> generates both server and client code. Use <code>GrpcServices="Server"</code> or <code>GrpcServices="Client"</code> to generate only what's needed.</div>
How do you implement a gRPC server in ASP.NET Core?	<div style="text-align:left"><code>// Generated base class from proto<br>using Grpc.Core;<br>using MyApp.Grpc;<br>public class GreeterService : Greeter.GreeterBase<br>{<br>    public override Task&lt;HelloReply&gt; SayHello(<br>        HelloRequest request,<br>        ServerCallContext context)<br>    {<br>        return Task.FromResult(new HelloReply<br>        {<br>            Message = $"Hello, {request.Name}!"<br>        });<br>    }<br>}<br>// Register in Program.cs<br>builder.Services.AddGrpc();<br>app.MapGrpcService&lt;GreeterService&gt;();</code><br><br>Senior insight: <code>ServerCallContext</code> gives access to metadata, deadlines, and cancellation tokens — always check <code>context.CancellationToken</code> for long-running operations.</div>
How do you create a gRPC client in .NET?	<div style="text-align:left"><code>// Factory pattern — recommended<br>builder.Services.AddGrpcClient&lt;Greeter.GreeterClient&gt;(o =&gt;<br>{<br>    o.Address = new Uri("https://localhost:7042");<br>});<br>// Inject and use<br>public class MyService(Greeter.GreeterClient client)<br>{<br>    public async Task CallGreeter()<br>    {<br>        var reply = await client.SayHelloAsync(<br>            new HelloRequest { Name = "Alice" });<br>        Console.WriteLine(reply.Message);<br>    }<br>}</code><br><br>Senior insight: always use <code>AddGrpcClient</code> (the factory pattern) instead of creating channels manually — it integrates with DI, handles channel reuse, and supports HttpClientFactory retry/logging policies.</div>
How do you handle errors in gRPC .NET?	<div style="text-align:left">gRPC uses <code>RpcException</code> with <code>StatusCode</code> instead of HTTP status codes:<br><code>// Server — throw RpcException<br>public override Task&lt;UserReply&gt; GetUser(UserRequest req, ServerCallContext ctx)<br>{<br>    if (user is null)<br>        throw new RpcException(new Status(StatusCode.NotFound, "User not found"));<br>    return Task.FromResult(new UserReply { Name = user.Name });<br>}<br>// Client — catch RpcException<br>try {<br>    var reply = await client.GetUserAsync(new UserRequest { Id = 1 });<br>} catch (RpcException ex) when (ex.StatusCode == StatusCode.NotFound) {<br>    Console.WriteLine("Not found: " + ex.Status.Detail);<br>}</code><br>Common codes: <code>StatusCode.NotFound</code>, <code>StatusCode.InvalidArgument</code>, <code>StatusCode.Unauthenticated</code>, PermissionDenied, Internal, Unavailable.</div>
What are the 4 gRPC communication patterns and how are they defined in .proto?	<div style="text-align:left">1. Unary — one request, one response:<br><code>   rpc GetUser (Request) returns (Response);</code><br>2. Server streaming — one request, stream of responses:<br><code>   rpc ListUsers (Request) returns (stream Response);</code><br>3. Client streaming — stream of requests, one response:<br><code>   rpc UploadData (stream Request) returns (Response);</code><br>4. Bidirectional streaming — both sides stream:<br><code>   rpc Chat (stream Message) returns (stream Message);</code><br>In C# server streaming:<br><code>public override async Task ListUsers(Request req, IServerStreamWriter&lt;Response&gt; stream, ServerCallContext ctx) {<br>    foreach (var user in users)<br>        await stream.WriteAsync(new Response { Name = user.Name });<br>}</code></div>
How do you configure Kestrel to support gRPC and HTTP/2 in .NET?	<div style="text-align:left">gRPC requires HTTP/2. In production with TLS, this is automatic. For development without TLS:<br><code>// Program.cs<br>builder.WebHost.ConfigureKestrel(options =&gt;<br>{<br>    options.ListenLocalhost(5001, o =&gt; o.Protocols = HttpProtocols.Http2);<br>    options.ListenLocalhost(5000, o =&gt; o.Protocols = HttpProtocols.Http1); // for REST<br>});</code><br>Or in appsettings.json under Kestrel.Endpoints.<br><br>Senior insight: you can host gRPC and REST on different ports in the same process. <code>HttpProtocols.Http1</code> and <code>HttpProtocols.Http2</code> cannot share the same port without TLS ALPN negotiation.</div>
What is the difference between record, class, and struct in C# for data models?	<div style="text-align:left"><code>class</code>: reference type, heap-allocated, nullable, mutable by default. Use for complex objects with behavior.<br><code>record</code> (C# 9+): reference type, immutable by default, value-based equality, with-expression for copying. Best for DTOs and immutable data.<br><code>struct</code>: value type, stack-allocated, copied on assignment. Use for small, short-lived data (coordinates, RGB values).<br><code>// Record<br>public record UserDto(string Name, int Age);<br>var u2 = u1 with { Age = 31 }; // non-destructive mutation</code><br><br>Senior insight: use <code>record</code>s for API request/response DTOs — they're immutable, equality just works, and they deconstruct nicely.</div>
What are common C# mistakes when working with collections?	<div style="text-align:left">1. Modifying a list while iterating:<br><code>foreach (var item in list) list.Remove(item); // InvalidOperationException<br>// Fix: list.RemoveAll(x => condition);</code><br>2. Using <code>List&lt;T&gt;</code> for frequent <code>.Contains()</code> checks:<br><code>list.Contains(x) // O(n) — use HashSet&lt;T&gt; for O(1)</code><br>3. Confusing <code>.Count</code> (<code>List&lt;T&gt;</code>) with <code>.Length</code> (array):<br><code>nums.Length  // array<br>nums.Count   // List&lt;T&gt;</code><br>4. Not disposing <code>IEnumerable&lt;T&gt;</code> from LINQ:<br><code>var q = db.Users.Where(x => x.Active); // not executed yet<br>foreach (var u in q) ... // executes SQL here<br>// Calling q again re-executes the query!</code></div>
What is the difference between IEnumerable, ICollection, IList, and List&lt;T&gt;?	<div style="text-align:left"><code>IEnumerable&lt;T&gt;</code>: read-only, forward-only iteration. No <code>.Count</code>, no index access. Laziest — use as return type for flexibility.<br><code>ICollection&lt;T&gt;</code>: adds <code>.Count</code>, <code>.Add()</code>, <code>.Remove()</code>, <code>.Contains()</code>.<br><code>IList&lt;T&gt;</code>: adds index access (list[i]) and Insert.<br><code>List&lt;T&gt;</code>: concrete implementation of <code>IList&lt;T&gt;</code> with extra methods (Sort, Find, etc.).<br><br>Senior insight: return <code>IEnumerable&lt;T&gt;</code> from methods when callers only need to iterate — it lets you swap implementations (array, list, generator) without breaking callers. Accept the most specific type you need as parameters. Liskov substitution principle in practice.</div>
How does async/await work in C# and what is the difference from Python and JS?	<div style="text-align:left">C# <code>async</code>/<code>await</code> uses Task-based Asynchronous Pattern (TAP). An <code>async</code> method returns <code>Task</code> or <code>Task&lt;T&gt;</code> — a promise of future work.<br><code>async Task&lt;string&gt; FetchDataAsync() {<br>    var result = await httpClient.GetStringAsync(url);<br>    return result;<br>}</code><br>Comparisons:<br>- JS: single-threaded event loop, <code>async</code> is the only concurrency model<br>- Python asyncio: cooperative, single-threaded event loop<br>- C#: uses thread pool, true parallelism possible with <code>Task.WhenAll</code><br><br>Senior insight: in C#, <code>ConfigureAwait(false)</code> in library code avoids deadlocks by not capturing the synchronization context — always use it in non-UI library methods.</div>
What is the difference between Task.WhenAll and Task.WhenAny in C#?	<div style="text-align:left"><code>// WhenAll — wait for ALL tasks to complete (parallel)<br>var tasks = users.Select(u => SendEmailAsync(u));<br>await Task.WhenAll(tasks); // all run concurrently<br>// WhenAny — complete when FIRST task finishes<br>var winner = await Task.WhenAny(task1, task2, task3);</code><br><code>Task.WhenAll</code> is like Promise.all() in JS or asyncio.gather() in Python.<br><code>Task.WhenAny</code> is like Promise.race() in JS.<br><br>Senior insight: <code>Task.WhenAll</code> is the key to parallel I/O in C# — running 10 DB queries sequentially vs concurrently can be the difference between 1000ms and 100ms response time.</div>
What are common REST API best practices in ASP.NET Core?	<div style="text-align:left">1. Use <code>[ProducesResponseType]</code> to document all possible response codes<br>2. Return <code>IActionResult</code> or <code>ActionResult&lt;T&gt;</code> — not raw objects<br>3. Use ProblemDetails (RFC 7807) for error responses — built into ASP.NET Core<br>4. Implement versioning from day one: /api/v1/users<br>5. Use DTOs — never expose your domain/EF entities directly<br>6. Return 201 Created with Location header for POST<br>7. Use <code>CancellationToken</code> in all async endpoints:<br><code>public async Task&lt;IActionResult&gt; Get(CancellationToken ct)<br>    => Ok(await _repo.GetAllAsync(ct));</code><br><br>Senior insight: <code>CancellationToken</code> lets the server abort work when the client disconnects — critical for expensive DB queries.</div>
