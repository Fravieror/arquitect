What is a Python list and how does it differ from an array in C# or JavaScript?	A list in Python is a dynamic, ordered collection that can hold any mix of types: numbers, strings, objects, etc. Unlike C# arrays (fixed size, single type) or JS arrays (dynamic but single prototype chain), Python lists are truly flexible. Think of it as a resizable container where each slot can hold anything. Under the hood it's implemented as a dynamic array (like List<T> in C#). Example: my_list = [1, "hello", 3.14, True]

How do you create a for loop to iterate over a list in Python?	Use the `for item in list` syntax — the cleanest iteration in any mainstream language. Example:\nfor fruit in ["apple", "banana", "cherry"]:\n    print(fruit)\nNo index needed unless you explicitly want it. Senior insight: Python's for loop is actually a for-each — it calls __iter__ on the object. Everything iterable (lists, dicts, files, generators) works the same way.

How do you iterate a list with its index in Python?	Use enumerate() — never use range(len(list)), that's a junior mistake.\nfruits = ["apple", "banana", "cherry"]\nfor i, fruit in enumerate(fruits):\n    print(i, fruit)\nenumerate() returns (index, value) tuples. You can also start the index at 1: enumerate(fruits, start=1). In C# this is like doing a for loop with index; in JS it's like forEach with index parameter.

How do you iterate a list in reverse in Python?	Use reversed() or slice notation:\nfor item in reversed(my_list):\n    print(item)\nOr: for item in my_list[::-1]. reversed() is memory-efficient (returns an iterator), while [::-1] creates a new list copy. For large lists, prefer reversed(). In C# you'd use .Reverse() LINQ or iterate backwards with index.

How do you create a list with a range of numbers in Python?	Use range() — it's lazy (generates numbers on demand, not stored in memory):\nfor i in range(10):       # 0 to 9\nfor i in range(1, 11):    # 1 to 10\nfor i in range(0, 10, 2): # 0, 2, 4, 6, 8 (step)\nTo get an actual list: nums = list(range(10)). Senior insight: range() is an iterable, not a list — it doesn't allocate memory for all elements upfront. Much more efficient than JS's Array.from({length: 10}, (_, i) => i).

What is a list comprehension in Python and when should you use it?	A concise way to create a new list by transforming or filtering another iterable:\nsquares = [x**2 for x in range(10)]\nevens = [x for x in range(20) if x % 2 == 0]\nEquivalent to map/filter in JS or LINQ Select/Where in C#. Use when the logic is simple and fits one line. Avoid nesting more than 2 levels — it becomes unreadable. Senior insight: list comprehensions are faster than equivalent for loops because they're optimized at the CPython bytecode level.

What is a Python set (hashset) and what problem does it solve?	A set is an unordered collection of unique values with O(1) average lookup, insertion, and deletion — backed by a hash table. Think of it as a dictionary with only keys and no values. Perfect for: checking membership fast, removing duplicates, set math (union, intersection). In C# it's HashSet<T>, in JS it's Set. Example: my_set = {1, 2, 3} or my_set = set([1, 2, 2, 3]) → {1, 2, 3}

How do you create and populate a set in Python?	# Empty set — MUST use set(), not {} (that creates a dict)\nmy_set = set()\n# From literal\nmy_set = {1, 2, 3}\n# From a list (removes duplicates)\nmy_set = set([1, 2, 2, 3, 3])  # → {1, 2, 3}\n# Add elements\nmy_set.add(4)\nmy_set.update([5, 6])  # add multiple\nCommon mistake: {} creates an empty dict, not a set. Always use set() for empty sets.

How do you iterate over a set in Python?	for item in my_set:\n    print(item)\nSets are unordered — you get no guaranteed order (insertion order is NOT preserved, unlike dicts since Python 3.7). Never rely on set iteration order. If you need order, sort first:\nfor item in sorted(my_set):\n    print(item)\nSenior insight: if order matters, reconsider whether a set is the right data structure.

How do you check membership in a set vs a list in Python?	# Set — O(1) average time\nif "apple" in my_set:  # ✅ fast\n# List — O(n) linear scan\nif "apple" in my_list:  # ⚠️ slow for large lists\nThis is the most important reason to use sets. If you're checking membership repeatedly against a fixed collection, convert the list to a set first. Real-world example: deduplication pipelines, permission checks, visited-node tracking in graph algorithms.

What are common set operations in Python?	a = {1, 2, 3}\nb = {2, 3, 4}\na | b   # union → {1, 2, 3, 4}     (C#: Union)\na & b   # intersection → {2, 3}    (C#: Intersect)\na - b   # difference → {1}          (C#: Except)\na ^ b   # symmetric diff → {1, 4}  (elements in one but not both)\nThese are O(n) operations. Extremely useful for comparing sets of IDs, permissions, tags, etc.

What is a Python dict (hashmap) and how does it compare to other languages?	A dict is an ordered (since Python 3.7) key-value store backed by a hash table. Keys must be hashable (immutable: strings, numbers, tuples). Values can be anything. Equivalent to: HashMap<K,V> in Java/C#, Map/object in JavaScript. Average O(1) for get, set, delete. Example:\nuser = {"name": "Alice", "age": 30, "active": True}

How do you create a dictionary in Python?	# Literal syntax\nconfig = {"host": "localhost", "port": 8080}\n# From pairs\nconfig = dict(host="localhost", port=8080)\n# Empty dict\nconfig = {}\n# From two lists (zip)\nkeys = ["a", "b", "c"]\nvalues = [1, 2, 3]\nresult = dict(zip(keys, values))  # {"a": 1, "b": 2, "c": 3}\nDict comprehension:\nsquares = {x: x**2 for x in range(5)}

How do you safely access a value in a Python dict?	# Direct access — raises KeyError if key missing\nvalue = my_dict["key"]  # ⚠️ dangerous\n# Safe access with default\nvalue = my_dict.get("key")           # returns None if missing\nvalue = my_dict.get("key", "default")  # returns "default"\n# Check first\nif "key" in my_dict:\n    value = my_dict["key"]\nSenior insight: prefer .get() over direct access unless you explicitly want to crash on a missing key (fail-fast design).

How do you iterate over a dictionary in Python?	d = {"a": 1, "b": 2, "c": 3}\n# Keys only (default)\nfor key in d:\n    print(key)\n# Values only\nfor value in d.values():\n    print(value)\n# Both — most common\nfor key, value in d.items():\n    print(key, value)\nSenior insight: .items() returns view objects — they reflect changes to the dict in real-time and don't copy data. Don't modify the dict while iterating — it raises RuntimeError.

How do you merge two dictionaries in Python?	# Python 3.9+ (recommended)\nmerged = dict1 | dict2\n# Python 3.5+\nmerged = {**dict1, **dict2}\n# In-place update (modifies dict1)\ndict1.update(dict2)\nWhen keys conflict, the rightmost dict wins. Senior insight: the | operator is the cleanest and most readable. Use {**a, **b} for older codebases. Never use loops to merge dicts — it's unnecessary complexity.

What is defaultdict and when should you use it?	from collections import defaultdict\n# Automatically creates a default value for missing keys\ncounts = defaultdict(int)   # missing keys default to 0\ngrouped = defaultdict(list) # missing keys default to []\ncounts["visits"] += 1  # no KeyError even though key didn't exist\ngrouped["users"].append("Alice")\nPerfect for: frequency counters, grouping items, building adjacency lists in graphs. Avoids the verbose: if key not in d: d[key] = [] pattern.

What is a Python virtual environment and why is it essential?	A virtual environment (venv) is an isolated Python installation for a specific project — it has its own Python binary and packages, separate from the system Python. Think of it like Docker but just for Python packages. Without it, all projects share one global package space — version conflicts become inevitable (Project A needs requests 2.x, Project B needs requests 3.x). Every Python project should have its own venv.

How do you create and activate a virtual environment in Python?	# Create\npython -m venv venv\n# Activate — Linux/Mac\nsource venv/bin/activate\n# Activate — Windows CMD\nvenv\\Scripts\\activate.bat\n# Activate — Windows PowerShell\nvenv\\Scripts\\Activate.ps1\n# Verify (should show venv path)\nwhich python  # or: where python on Windows\n# Deactivate\ndeactivate\nConvention: name the folder "venv" or ".venv" and add it to .gitignore.

What is the difference between pip install and requirements.txt?	pip install package installs a package into the active venv. requirements.txt is a file listing all dependencies with pinned versions:\nrequests==2.31.0\nfastapi==0.110.0\n# Generate from current env:\npip freeze > requirements.txt\n# Install from file:\npip install -r requirements.txt\nSenior insight: pip freeze pins EVERY transitive dependency — this is great for reproducibility but noisy. Prefer tools like pip-tools or poetry which separate direct vs transitive deps.

What tools exist beyond basic venv for Python environment management?	poetry — manages deps + venv + publishing, uses pyproject.toml. The modern standard for new projects.\npipenv — combines pip + venv, uses Pipfile. Less popular now.\npyenv — manages multiple Python versions (like nvm for Node).\nconda — popular in data science, manages non-Python deps too.\nuv — new ultra-fast package manager written in Rust (2024). Very fast.\nSenior insight: for backend services, poetry or uv are the current best practices. For DS/ML, conda is common.

What is FastAPI and what makes it different from Flask or Django?	FastAPI is a modern async Python web framework built on Starlette and Pydantic. Key differentiators:\n1. Async-first: built on ASGI, handles concurrent requests efficiently\n2. Auto validation: uses Pydantic models to validate request/response data\n3. Auto docs: generates OpenAPI (Swagger) and ReDoc docs automatically\n4. Type hints: uses Python type annotations for validation and IDE support\nFlask: minimal, sync, no built-in validation. Django: full-stack, ORM, batteries-included. FastAPI: API-focused, async, modern.

How do you create a minimal FastAPI application?	from fastapi import FastAPI\n\napp = FastAPI()\n\n@app.get("/")\ndef read_root():\n    return {"message": "Hello, World!"}\n\n@app.get("/items/{item_id}")\ndef read_item(item_id: int, q: str = None):\n    return {"item_id": item_id, "q": q}\n# Run with:\n# uvicorn main:app --reload\nThe type annotations (item_id: int) are not just hints — FastAPI uses them to validate and coerce input. If item_id is not an integer, FastAPI returns a 422 error automatically.

How do you run a FastAPI application and what is uvicorn?	uvicorn is an ASGI server (like Kestrel for .NET or Gunicorn for WSGI). ASGI is the async version of WSGI.\n# Development (with auto-reload)\nuvicorn main:app --reload\n# Production (multiple workers)\nuvicorn main:app --workers 4 --host 0.0.0.0 --port 8000\n# Or with gunicorn as process manager\ngunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker\nSenior insight: in production, always use multiple workers + a reverse proxy (nginx). Never expose uvicorn directly to the internet.

How do you define request/response models in FastAPI using Pydantic?	from fastapi import FastAPI\nfrom pydantic import BaseModel\n\nclass UserCreate(BaseModel):\n    name: str\n    age: int\n    email: str\n\napp = FastAPI()\n\n@app.post("/users")\ndef create_user(user: UserCreate):\n    return {"created": user.name}\nPydantic validates the incoming JSON body against UserCreate. If "age" is "abc", it returns a 422 with details. Think of it as ASP.NET's model binding + DataAnnotations combined, but more powerful.

How do you handle path parameters, query params, and request body in FastAPI?	from fastapi import FastAPI\nfrom pydantic import BaseModel\n\napp = FastAPI()\n\nclass Item(BaseModel):\n    name: str\n    price: float\n\n@app.put("/items/{item_id}")\ndef update_item(\n    item_id: int,         # path parameter\n    q: str = None,        # query parameter (optional)\n    item: Item = None     # request body\n):\n    return {"item_id": item_id, "q": q, "name": item.name}\nFastAPI infers the source: path params match URL segments, simple types become query params, Pydantic models become request body.

How do you use async endpoints in FastAPI?	from fastapi import FastAPI\nimport httpx\n\napp = FastAPI()\n\n@app.get("/data")\nasync def get_data():\n    async with httpx.AsyncClient() as client:\n        response = await client.get("https://api.example.com/data")\n        return response.json()\nUse async def when your endpoint does I/O (DB calls, HTTP requests, file reads). Use regular def for CPU-bound or sync code. Senior insight: mixing async and sync incorrectly is a common bug — calling a sync blocking function inside async def blocks the entire event loop.

How do you add dependency injection in FastAPI?	from fastapi import FastAPI, Depends\n\ndef get_db():\n    db = connect_to_db()\n    try:\n        yield db\n    finally:\n        db.close()\n\napp = FastAPI()\n\n@app.get("/users")\ndef list_users(db=Depends(get_db)):\n    return db.query_all_users()\nDepends() is FastAPI's DI system. Works for: DB sessions, auth, shared config, rate limiting. Supports async, caching, nested deps. Equivalent to .NET's constructor injection but declared inline.

What is gRPC and when should you use it instead of REST?	gRPC is a high-performance RPC framework using Protocol Buffers (binary serialization) over HTTP/2. Use gRPC when:\n- Services talk to each other (internal microservices)\n- You need high throughput and low latency\n- You want strongly-typed contracts enforced at compile time\n- Streaming (server push, client streaming) is needed\nUse REST when: public APIs, browser clients, simple CRUD, team unfamiliar with Protobuf.\nSenior insight: gRPC is ~5-10x faster than JSON REST for internal service communication due to binary encoding and HTTP/2 multiplexing.

What is a .proto file and what role does it play in gRPC?	A .proto file defines the service contract in Protocol Buffer language — it specifies services (methods) and message types (data shapes). It's language-agnostic: the same .proto generates client/server code in Python, Go, C#, Java, etc.\nExample:\nsyntax = "proto3";\npackage greeter;\nservice Greeter {\n  rpc SayHello (HelloRequest) returns (HelloReply);\n}\nmessage HelloRequest { string name = 1; }\nmessage HelloReply { string message = 1; }\nThink of it as an IDL (Interface Definition Language) — like a contract that all parties must follow.

How do you generate Python gRPC code from a .proto file?	Install tools:\npip install grpcio grpcio-tools\nGenerate Python files:\npython -m grpc_tools.protoc \\\n  -I. \\\n  --python_out=. \\\n  --grpc_python_out=. \\\n  greeter.proto\nThis generates two files:\n- greeter_pb2.py: message classes (data)\n- greeter_pb2_grpc.py: server stubs + client stubs\nSenior insight: commit generated files or generate them in CI — never edit them manually. Treat them like migration files.

How do you create a gRPC server in Python?	import grpc\nfrom concurrent import futures\nimport greeter_pb2\nimport greeter_pb2_grpc\n\nclass GreeterServicer(greeter_pb2_grpc.GreeterServicer):\n    def SayHello(self, request, context):\n        return greeter_pb2.HelloReply(\n            message=f"Hello, {request.name}!"\n        )\n\ndef serve():\n    server = grpc.server(futures.ThreadPoolExecutor(max_workers=10))\n    greeter_pb2_grpc.add_GreeterServicer_to_server(GreeterServicer(), server)\n    server.add_insecure_port("[::]:50051")\n    server.start()\n    server.wait_for_termination()\n\nif __name__ == "__main__":\n    serve()

How do you create a gRPC client in Python?	import grpc\nimport greeter_pb2\nimport greeter_pb2_grpc\n\ndef run():\n    # Create a channel to the server\n    with grpc.insecure_channel("localhost:50051") as channel:\n        # Create a stub (client)\n        stub = greeter_pb2_grpc.GreeterStub(channel)\n        # Call the remote method like a local function\n        response = stub.SayHello(\n            greeter_pb2.HelloRequest(name="Alice")\n        )\n        print(response.message)  # "Hello, Alice!"\n\nif __name__ == "__main__":\n    run()\nThe stub makes remote calls look like local function calls — that's the whole point of RPC.

What is the difference between insecure and secure gRPC channels?	# Insecure — no TLS, development only\nchannel = grpc.insecure_channel("localhost:50051")\n# Secure — TLS, required for production\ncredentials = grpc.ssl_channel_credentials(\n    root_certificates=open("ca.crt", "rb").read()\n)\nchannel = grpc.secure_channel("api.example.com:443", credentials)\nNever use insecure channels in production. For mutual TLS (mTLS — both client and server authenticate each other), use grpc.ssl_channel_credentials() with client cert and key. Senior insight: mTLS is the standard for zero-trust service meshes (Istio, Linkerd).

What are the 4 types of gRPC communication patterns?	1. Unary RPC — one request, one response (like REST)\n   rpc GetUser(Request) returns (Response)\n2. Server streaming — one request, stream of responses\n   rpc ListUsers(Request) returns (stream Response)\n3. Client streaming — stream of requests, one response\n   rpc UploadData(stream Request) returns (Response)\n4. Bidirectional streaming — both sides stream simultaneously\n   rpc Chat(stream Message) returns (stream Message)\nSenior insight: bidirectional streaming enables real-time features (live updates, collaborative editing) far more efficiently than WebSockets + REST polling.

How do you handle errors in gRPC Python?	gRPC uses status codes instead of HTTP codes:\nimport grpc\n\n# Server side — set error status\ndef GetUser(self, request, context):\n    if not found:\n        context.set_code(grpc.StatusCode.NOT_FOUND)\n        context.set_details("User not found")\n        return UserReply()\n\n# Client side — catch error\ntry:\n    response = stub.GetUser(request)\nexcept grpc.RpcError as e:\n    print(e.code())     # StatusCode.NOT_FOUND\n    print(e.details())  # "User not found"\nCommon codes: NOT_FOUND, INVALID_ARGUMENT, UNAUTHENTICATED, PERMISSION_DENIED, INTERNAL.

What are common Python mistakes when working with lists and dicts?	1. Mutable default arguments:\ndef add(item, lst=[]):  # ⚠️ BAD — shared across calls!\n    lst.append(item); return lst\ndef add(item, lst=None): lst = lst or []; ...  # ✅\n2. Modifying a list while iterating it — use a copy:\nfor item in my_list[:]:  # iterate copy\n3. Using list as a stack pop from front:\nmy_list.pop(0)  # O(n) — use collections.deque instead\n4. Checking key existence with try/except vs .get():\n# Prefer .get() for expected missing keys; try/except for truly exceptional cases

What is the GIL and how does it affect Python concurrency?	The GIL (Global Interpreter Lock) is a mutex in CPython that allows only one thread to execute Python bytecode at a time. This means:\n- Threading: good for I/O-bound tasks (threads release GIL during I/O)\n- Threading: BAD for CPU-bound tasks (threads block each other)\n- Multiprocessing: bypasses GIL entirely (separate processes)\n- asyncio: single-threaded, cooperative concurrency for I/O\nSenior insight: FastAPI with async endpoints handles thousands of concurrent requests on a single thread via the event loop — the GIL is irrelevant there. For CPU-heavy work, use ProcessPoolExecutor.

How does Python's async/await compare to C# and JavaScript?	All three use the same async/await keywords and event loop concept, but with differences:\nPython: asyncio event loop, must use async-compatible libraries (httpx not requests, asyncpg not psycopg2)\nJS: single-threaded by design, all I/O is inherently async\nC#: Task-based, async/await integrates deeply with .NET runtime\nPython gotcha: mixing sync and async is easy to get wrong. Calling a blocking function (time.sleep, requests.get) in an async def blocks the entire event loop. Always use: await asyncio.sleep(), await httpx...\nUse asyncio.run_in_executor() to run sync blocking code from async context.

What is the difference between __repr__ and __str__ in Python classes?	__str__: human-readable string, called by str() and print()\n__repr__: unambiguous developer representation, called by repr() and in the REPL\nclass Point:\n    def __init__(self, x, y): self.x, self.y = x, y\n    def __str__(self): return f"Point({self.x}, {self.y})"\n    def __repr__(self): return f"Point(x={self.x!r}, y={self.y!r})"\nRule: __repr__ should ideally be valid Python that recreates the object. If only one is defined, __repr__ is used as fallback for __str__. Senior insight: always implement __repr__ in your domain models — it saves hours of debugging.

What is type hinting in Python and why does it matter at the senior level?	Type hints (PEP 484) are annotations that specify expected types. They don't enforce types at runtime (Python is still dynamic) but:\n- Enable IDE autocompletion and error detection\n- Used by FastAPI/Pydantic for runtime validation\n- Improve code readability and maintainability\n- Allow tools like mypy to do static type checking\nfrom typing import Optional, List, Dict\ndef get_users(ids: List[int]) -> Optional[Dict[str, str]]:\n    ...\nSenior insight: untyped Python in a large codebase is a maintenance nightmare. Always type hint public APIs and function signatures.

What are Python dataclasses and when to use them vs Pydantic?	from dataclasses import dataclass\n\n@dataclass\nclass Point:\n    x: float\n    y: float\n    label: str = "origin"\n\np = Point(1.0, 2.0)\nDataclasses auto-generate __init__, __repr__, __eq__. Use when:\n- Simple data containers with no validation needed\n- No JSON serialization needed\n- Performance matters (faster than Pydantic)\nUse Pydantic when:\n- You need validation, coercion, JSON serialization\n- Working with FastAPI or external data (user input, API responses)\nSenior insight: dataclasses are stdlib; Pydantic is a dependency. Choose based on requirements.
