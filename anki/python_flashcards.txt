What is a Python list and how does it differ from an array in C# or JavaScript?	<div style="text-align:left">A list in Python is a dynamic, ordered collection that can hold any mix of types: numbers, strings, objects, etc. Unlike C# arrays (fixed size, single type) or JS arrays (dynamic but single prototype chain), Python lists are truly flexible. Think of it as a resizable container where each slot can hold anything. Under the hood it's implemented as a dynamic array (like List&lt;T&gt; in C#).<br>Example: <code>my_list = [1, "hello", 3.14, True]</code></div>
How do you create a for loop to iterate over a list in Python?	<div style="text-align:left">Use the <code>for item in list</code> syntax — the cleanest iteration in any mainstream language.<br>Example:<br><code>for fruit in ["apple", "banana", "cherry"]:<br>    print(fruit)</code><br>No index needed unless you explicitly want it.<br><br>Senior insight: Python's <code>for</code> loop is actually a for-each — it calls <code>__iter__</code> on the object. Everything iterable (lists, dicts, files, generators) works the same way.</div>
How do you iterate a list with its index in Python?	<div style="text-align:left">Use <code>enumerate()</code> — never use <code>range(len(list))</code>, that's a junior mistake.<br><code>fruits = ["apple", "banana", "cherry"]<br>for i, fruit in enumerate(fruits):<br>    print(i, fruit)</code><br><code>enumerate()</code> returns (index, value) tuples. You can also start the index at 1: <code>enumerate(fruits, start=1)</code>. In C# this is like doing a <code>for</code> loop with index; <code>in</code> JS it's like forEach with index parameter.</div>
How do you iterate a list in reverse in Python?	<div style="text-align:left">Use <code>reversed()</code> or slice notation:<br><code>for item in reversed(my_list):<br>    print(item)</code><br>Or: <code>for item in my_list[::-1]</code>. <code>reversed()</code> is memory-efficient (returns an iterator), while <code>[::-1]</code> creates a new list copy. For large lists, prefer <code>reversed()</code>. In C# you'd use .Reverse() LINQ or iterate backwards with index.</div>
How do you create a list with a range of numbers in Python?	<div style="text-align:left">Use <code>range()</code> — it's lazy (generates numbers on demand, <code>not</code> stored <code>in</code> memory):<br><code>for i in range(10):       # 0 to 9<br>for i in range(1, 11):    # 1 to 10<br>for i in range(0, 10, 2): # 0, 2, 4, 6, 8 (step)</code><br>To get an actual list: <code>nums = list(range(10))</code>.<br><br>Senior insight: <code>range()</code> is an iterable, <code>not</code> a list — it doesn't allocate memory for all elements upfront. Much more efficient than JS's <code>Array.from({length: 10}, (_, i) => i)</code>.</div>
What is a list comprehension in Python and when should you use it?	<div style="text-align:left">A concise way to create a new list by transforming or filtering another iterable:<br><code>squares = [x**2 for x in range(10)]<br>evens = [x for x in range(20) if x % 2 == 0]</code><br>Equivalent to <code>map</code>/<code>filter</code> <code>in</code> JS or LINQ Select/Where <code>in</code> C#. Use when the logic is simple <code>and</code> fits one line. Avoid nesting more than 2 levels — it becomes unreadable.<br><br>Senior insight: list comprehensions are faster than equivalent <code>for</code> loops because they're optimized at the CPython bytecode level.</div>
What is a Python set (hashset) and what problem does it solve?	<div style="text-align:left">A <code>set()</code> is an unordered collection of unique values with O(1) average lookup, insertion, <code>and</code> deletion — backed by a hash table. Think of it as a dictionary with only keys <code>and</code> no values. Perfect for: checking membership fast, removing duplicates, set math (union, intersection). <code>in</code> C# it's HashSet&lt;T&gt;, <code>in</code> JS it's Set.<br>Example: <code>my_set = {1, 2, 3}</code> or <code>my_set = set([1, 2, 2, 3])</code> → {1, 2, 3}</div>
How do you create and populate a set in Python?	<div style="text-align:left"><code># Empty set — MUST use set(), not {} (that creates a dict)<br>my_set = set()<br># From literal<br>my_set = {1, 2, 3}<br># From a list (removes duplicates)<br>my_set = set([1, 2, 2, 3, 3])  # → {1, 2, 3}<br># Add elements<br>my_set.add(4)<br>my_set.update([5, 6])  # add multiple</code><br><br>Common mistake: <code>{}</code> creates an empty dict, <code>not</code> a <code>set()</code>. Always use <code>set()</code> for empty sets.</div>
How do you iterate over a set in Python?	<div style="text-align:left"><code>for item in my_set:<br>    print(item)</code><br>Sets are unordered — you get no guaranteed order (insertion order is <code>not</code> preserved, unlike dicts since Python 3.7). Never rely on set iteration order. If you need order, sort first:<br><code>for item in sorted(my_set):<br>    print(item)</code><br><br>Senior insight: if order matters, reconsider whether a set is the right data structure.</div>
How do you check membership in a set vs a list in Python?	<div style="text-align:left"><code># Set — O(1) average time<br>if "apple" in my_set:  # ✅ fast<br># List — O(n) linear scan<br>if "apple" in my_list:  # ⚠️ slow for large lists</code><br>This is the most important reason to use sets. If you're checking membership repeatedly against a fixed collection, convert the list to a set first. Real-world example: deduplication pipelines, permission checks, visited-node tracking <code>in</code> graph algorithms.</div>
What are common set operations in Python?	<div style="text-align:left"><code>a = {1, 2, 3}<br>b = {2, 3, 4}<br>a | b   # union → {1, 2, 3, 4}     (C#: Union)<br>a & b   # intersection → {2, 3}    (C#: Intersect)<br>a - b   # difference → {1}          (C#: Except)<br>a ^ b   # symmetric diff → {1, 4}  (elements in one but not both)</code><br>These are O(n) operations. Extremely useful for comparing sets of IDs, permissions, tags, etc.</div>
What is a Python dict (hashmap) and how does it compare to other languages?	<div style="text-align:left">A <code>dict()</code> is an ordered (since Python 3.7) key-value store backed by a hash table. Keys must be hashable (immutable: strings, numbers, tuples). Values can be anything. Equivalent to: HashMap&lt;K,V&gt; <code>in</code> Java/C#, Map/object <code>in</code> JavaScript. Average O(1) for get, set, delete.<br>Example:<br><code>user = {"name": "Alice", "age": 30, "active": True}</code></div>
How do you create a dictionary in Python?	<div style="text-align:left"><code># Literal syntax<br>config = {"host": "localhost", "port": 8080}<br># From pairs<br>config = dict(host="localhost", port=8080)<br># Empty dict<br>config = {}<br># From two lists (zip)<br>keys = ["a", "b", "c"]<br>values = [1, 2, 3]<br>result = dict(zip(keys, values))  # {"a": 1, "b": 2, "c": 3}<br># Dict comprehension:<br>squares = {x: x**2 for x in range(5)}</code></div>
How do you safely access a value in a Python dict?	<div style="text-align:left"><code># Direct access — raises KeyError if key missing<br>value = my_dict["key"]  # ⚠️ dangerous<br># Safe access with default<br>value = my_dict.get("key")           # returns None if missing<br>value = my_dict.get("key", "default")  # returns "default"<br># Check first<br>if "key" in my_dict:<br>    value = my_dict["key"]</code><br><br>Senior insight: prefer <code>.get()</code> over direct access unless you explicitly want to crash on a missing key (fail-fast design).</div>
How do you iterate over a dictionary in Python?	<div style="text-align:left"><code>d = {"a": 1, "b": 2, "c": 3}<br># Keys only (default)<br>for key in d:<br>    print(key)<br># Values only<br>for value in d.values():<br>    print(value)<br># Both — most common<br>for key, value in d.items():<br>    print(key, value)</code><br><br>Senior insight: <code>.items()</code> returns view objects — they reflect changes to the dict <code>in</code> real-time <code>and</code> don't copy data. Don't modify the dict while iterating — it raises <code>RuntimeError</code>.</div>
How do you merge two dictionaries in Python?	<div style="text-align:left"><code># Python 3.9+ (recommended)<br>merged = dict1 | dict2<br># Python 3.5+<br>merged = {**dict1, **dict2}<br># In-place update (modifies dict1)<br>dict1.update(dict2)</code><br>When keys conflict, the rightmost dict wins.<br><br>Senior insight: the <code>|</code> operator is the cleanest <code>and</code> most readable. Use <code>{**a, **b}</code> for older codebases. Never use loops to merge dicts — it's unnecessary complexity.</div>
What is defaultdict and when should you use it?	<div style="text-align:left"><code>from collections import defaultdict<br># Automatically creates a default value for missing keys<br>counts = defaultdict(int)   # missing keys default to 0<br>grouped = defaultdict(list) # missing keys default to []<br>counts["visits"] += 1  # no KeyError even though key didn't exist<br>grouped["users"].append("Alice")</code><br>Perfect for: frequency counters, grouping items, building adjacency lists <code>in</code> graphs. Avoids the verbose: <code>if key not in d: d[key] = []</code> pattern.</div>
What is a Python virtual environment and why is it essential?	<div style="text-align:left">A virtual environment (<code>venv</code>) is an isolated Python installation for a specific project — it has its own Python binary <code>and</code> packages, separate from the system Python. Think of it like Docker but just for Python packages. Without it, all projects share one global package space — version conflicts become inevitable (Project A needs requests 2.x, Project B needs requests 3.x). Every Python project should have its own <code>venv</code>.</div>
How do you create and activate a virtual environment in Python?	<div style="text-align:left"><code># Create<br>python -m venv venv<br># Activate — Linux/Mac<br>source venv/bin/activate<br># Activate — Windows CMD<br>venv\Scripts\activate.bat<br># Activate — Windows PowerShell<br>venv\Scripts\Activate.ps1<br># Verify (should show venv path)<br>which python  # or: where python on Windows<br># Deactivate<br>deactivate</code><br><br>Convention: name the folder "venv" or ".venv" <code>and</code> add it to .gitignore.</div>
What is the difference between pip install and requirements.txt?	<div style="text-align:left"><code>pip install package</code> installs a package into the active <code>venv</code>. requirements.txt is a file listing all dependencies with pinned versions:<br><code>requests==2.31.0<br>fastapi==0.110.0<br># Generate from current env:<br>pip freeze > requirements.txt<br># Install from file:<br>pip install -r requirements.txt</code><br><br>Senior insight: <code>pip freeze</code> pins EVERY transitive dependency — this is great for reproducibility but noisy. Prefer tools like pip-tools or <code>poetry</code> which separate direct vs transitive deps.</div>
What tools exist beyond basic venv for Python environment management?	<div style="text-align:left"><code>poetry</code> — manages deps + <code>venv</code> + publishing, uses pyproject.toml. The modern standard for new projects.<br><code>pipenv</code> — combines <code>pip install</code> + <code>venv</code>, uses Pipfile. Less popular now.<br><code>pyenv</code> — manages multiple Python versions (like nvm for Node).<br><code>conda</code> — popular <code>in</code> data science, manages non-Python deps too.<br>uv — new ultra-fast package manager written <code>in</code> Rust (2024). Very fast.<br><br>Senior insight: for backend services, <code>poetry</code> or uv are the current best practices. For DS/ML, <code>conda</code> is common.</div>
What is FastAPI and what makes it different from Flask or Django?	<div style="text-align:left"><code>FastAPI</code> is a modern <code>async</code> Python web framework built on Starlette <code>and</code> <code>pydantic</code>. Key differentiators:<br>1. Async-first: built on ASGI, handles concurrent requests efficiently<br>2. Auto validation: uses <code>pydantic</code> models to validate request/response data<br>3. Auto docs: generates OpenAPI (Swagger) <code>and</code> ReDoc docs automatically<br>4. Type hints: uses Python type annotations for validation <code>and</code> IDE support<br>Flask: minimal, sync, no built-<code>in</code> validation. Django: full-stack, ORM, batteries-included. <code>FastAPI</code>: API-focused, <code>async</code>, modern.</div>
How do you create a minimal FastAPI application?	<div style="text-align:left"><code>from fastapi import FastAPI<br><br>app = FastAPI()<br><br>@app.get("/")<br>def read_root():<br>    return {"message": "Hello, World!"}<br><br>@app.get("/items/{item_id}")<br>def read_item(item_id: int, q: str = None):<br>    return {"item_id": item_id, "q": q}<br># Run with:<br># uvicorn main:app --reload</code><br>The type annotations (<code>item_id: int</code>) are <code>not</code> just hints — <code>FastAPI</code> uses them to validate <code>and</code> coerce input. If <code>item_id</code> is <code>not</code> an integer, <code>FastAPI</code> returns a 422 error automatically.</div>
How do you run a FastAPI application and what is uvicorn?	<div style="text-align:left"><code>uvicorn</code> is an ASGI server (like Kestrel for .NET or Gunicorn for WSGI). ASGI is the <code>async</code> version of WSGI.<br><code># Development (with auto-reload)<br>uvicorn main:app --reload<br># Production (multiple workers)<br>uvicorn main:app --workers 4 --host 0.0.0.0 --port 8000<br># Or with gunicorn as process manager<br>gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker</code><br><br>Senior insight: <code>in</code> production, always use multiple workers + a reverse proxy (nginx). Never expose <code>uvicorn</code> directly to the internet.</div>
How do you define request/response models in FastAPI using Pydantic?	<div style="text-align:left"><code>from fastapi import FastAPI<br>from pydantic import BaseModel<br><br>class UserCreate(BaseModel):<br>    name: str<br>    age: int<br>    email: str<br><br>app = FastAPI()<br><br>@app.post("/users")<br>def create_user(user: UserCreate):<br>    return {"created": user.name}</code><br><code>pydantic</code> validates the incoming JSON body against <code>UserCreate</code>. If "age" is "abc", it returns a 422 with details. Think of it as ASP.NET's model binding + DataAnnotations combined, but more powerful.</div>
How do you handle path parameters, query params, and request body in FastAPI?	<div style="text-align:left"><code>from fastapi import FastAPI<br>from pydantic import BaseModel<br><br>app = FastAPI()<br><br>class Item(BaseModel):<br>    name: str<br>    price: float<br><br>@app.put("/items/{item_id}")<br>def update_item(<br>    item_id: int,         # path parameter<br>    q: str = None,        # query parameter (optional)<br>    item: Item = None     # request body<br>):<br>    return {"item_id": item_id, "q": q, "name": item.name}</code><br><code>FastAPI</code> infers the source: path params match URL segments, simple types become query params, <code>pydantic</code> models become request body.</div>
How do you use async endpoints in FastAPI?	<div style="text-align:left"><code>from fastapi import FastAPI<br>import httpx<br><br>app = FastAPI()<br><br>@app.get("/data")<br>async def get_data():<br>    async with httpx.AsyncClient() as client:<br>        response = await client.get("https://api.example.com/data")<br>        return response.json()</code><br>Use <code>async def</code> when your endpoint does I/O (DB calls, HTTP requests, file reads). Use regular <code>def</code> for CPU-bound or sync code.<br><br>Senior insight: mixing <code>async</code> <code>and</code> sync incorrectly is a common bug — calling a sync blocking function inside <code>async def</code> blocks the entire event loop.</div>
How do you add dependency injection in FastAPI?	<div style="text-align:left"><code>from fastapi import FastAPI, Depends<br><br>def get_db():<br>    db = connect_to_db()<br>    try:<br>        yield db<br>    finally:<br>        db.close()<br><br>app = FastAPI()<br><br>@app.get("/users")<br>def list_users(db=Depends(get_db)):<br>    return db.query_all_users()</code><br><code>Depends()</code> is <code>FastAPI</code>'s DI system. Works for: DB sessions, auth, shared config, rate limiting. Supports <code>async</code>, caching, nested deps. Equivalent to .NET's constructor injection but declared inline.</div>
What is gRPC and when should you use it instead of REST?	<div style="text-align:left">gRPC is a high-performance RPC framework using Protocol Buffers (binary serialization) over HTTP/2. Use gRPC when:<br>- Services talk to each other (internal microservices)<br>- You need high throughput <code>and</code> low latency<br>- You want strongly-typed contracts enforced at compile time<br>- Streaming (server push, client streaming) is needed<br>Use REST when: public APIs, browser clients, simple CRUD, team unfamiliar with Protobuf.<br><br>Senior insight: gRPC is ~5-10x faster than JSON REST for internal service communication due to binary encoding <code>and</code> HTTP/2 multiplexing.</div>
What is a .proto file and what role does it play in gRPC?	<div style="text-align:left">A .proto file defines the service contract <code>in</code> Protocol Buffer language — it specifies services (methods) <code>and</code> message types (data shapes). It's language-agnostic: the same .proto generates client/server code <code>in</code> Python, Go, C#, Java, etc.<br>Example:<br><code>syntax = "proto3";<br>package greeter;<br>service Greeter {<br>  rpc SayHello (HelloRequest) returns (HelloReply);<br>}<br>message HelloRequest { string name = 1; }<br>message HelloReply { string message = 1; }</code><br>Think of it as an IDL (Interface Definition Language) — like a contract that all parties must follow.</div>
How do you generate Python gRPC code from a .proto file?	<div style="text-align:left">Install tools:<br><code>pip install grpcio grpcio-tools</code><br>Generate Python files:<br><code>python -m grpc_tools.protoc \<br>  -I. \<br>  --python_out=. \<br>  --grpc_python_out=. \<br>  greeter.proto</code><br>This generates two files:<br>- greeter_pb2.py: message classes (data)<br>- greeter_pb2_grpc.py: server stubs + client stubs<br><br>Senior insight: commit generated files or generate them <code>in</code> CI — never edit them manually. Treat them like migration files.</div>
How do you create a gRPC server in Python?	<div style="text-align:left"><code>import grpc<br>from concurrent import futures<br>import greeter_pb2<br>import greeter_pb2_grpc<br><br>class GreeterServicer(greeter_pb2_grpc.GreeterServicer):<br>    def SayHello(self, request, context):<br>        return greeter_pb2.HelloReply(<br>            message=f"Hello, {request.name}!"<br>        )<br><br>def serve():<br>    server = grpc.server(futures.ThreadPoolExecutor(max_workers=10))<br>    greeter_pb2_grpc.add_GreeterServicer_to_server(GreeterServicer(), server)<br>    server.add_insecure_port("[::]:50051")<br>    server.start()<br>    server.wait_for_termination()<br><br>if __name__ == "__main__":<br>    serve()</code></div>
How do you create a gRPC client in Python?	<div style="text-align:left"><code>import grpc<br>import greeter_pb2<br>import greeter_pb2_grpc<br><br>def run():<br>    # Create a channel to the server<br>    with grpc.insecure_channel("localhost:50051") as channel:<br>        # Create a stub (client)<br>        stub = greeter_pb2_grpc.GreeterStub(channel)<br>        # Call the remote method like a local function<br>        response = stub.SayHello(<br>            greeter_pb2.HelloRequest(name="Alice")<br>        )<br>        print(response.message)  # "Hello, Alice!"<br><br>if __name__ == "__main__":<br>    run()</code><br>The <code>stub</code> makes remote calls look like local function calls — that's the whole point of RPC.</div>
What is the difference between insecure and secure gRPC channels?	<div style="text-align:left"><code># Insecure — no TLS, development only<br>channel = grpc.insecure_channel("localhost:50051")<br># Secure — TLS, required for production<br>credentials = grpc.ssl_channel_credentials(<br>    root_certificates=open("ca.crt", "rb").read()<br>)<br>channel = grpc.secure_channel("api.example.com:443", credentials)</code><br>Never use insecure channels <code>in</code> production. For mutual TLS (mTLS — both client <code>and</code> server authenticate each other), use <code>grpc.ssl_channel_credentials()</code> with client cert <code>and</code> key.<br><br>Senior insight: mTLS is the standard for zero-trust service meshes (Istio, Linkerd).</div>
What are the 4 types of gRPC communication patterns?	<div style="text-align:left">1. Unary RPC — one request, one response (like REST)<br>   <code>rpc GetUser(Request) returns (Response)</code><br>2. Server streaming — one request, stream of responses<br>   <code>rpc ListUsers(Request) returns (stream Response)</code><br>3. Client streaming — stream of requests, one response<br>   <code>rpc UploadData(stream Request) returns (Response)</code><br>4. Bidirectional streaming — both sides stream simultaneously<br>   <code>rpc Chat(stream Message) returns (stream Message)</code><br><br>Senior insight: bidirectional streaming enables real-time features (live updates, collaborative editing) far more efficiently than WebSockets + REST polling.</div>
How do you handle errors in gRPC Python?	<div style="text-align:left">gRPC uses status codes instead of HTTP codes:<br><code>import grpc<br><br># Server side — set error status<br>def GetUser(self, request, context):<br>    if not found:<br>        context.set_code(grpc.StatusCode.NOT_FOUND)<br>        context.set_details("User not found")<br>        return UserReply()<br><br># Client side — catch error<br>try:<br>    response = stub.GetUser(request)<br>except grpc.RpcError as e:<br>    print(e.code())     # StatusCode.NOT_FOUND<br>    print(e.details())  # "User not found"</code><br>Common codes: NOT_FOUND, INVALID_ARGUMENT, UNAUTHENTICATED, PERMISSION_DENIED, INTERNAL.</div>
What are common Python mistakes when working with lists and dicts?	<div style="text-align:left">1. Mutable default arguments:<br><code>def add(item, lst=[]):  # ⚠️ BAD — shared across calls!<br>    lst.append(item); return lst<br>def add(item, lst=None): lst = lst or []; ...  # ✅</code><br>2. Modifying a list while iterating it — use a copy:<br><code>for item in my_list[:]:  # iterate copy</code><br>3. Using list as a stack <code>pop</code> from front:<br><code>my_list.pop(0)  # O(n) — use collections.deque instead</code><br>4. Checking key existence with try/except vs <code>.get()</code>:<br>Prefer <code>.get()</code> for expected missing keys; try/except for truly exceptional cases</div>
What is the GIL and how does it affect Python concurrency?	<div style="text-align:left">The GIL (Global Interpreter Lock) is a mutex <code>in</code> CPython that allows only one thread to execute Python bytecode at a time. This means:<br>- Threading: good for I/O-bound tasks (threads release GIL during I/O)<br>- Threading: BAD for CPU-bound tasks (threads block each other)<br>- Multiprocessing: bypasses GIL entirely (separate processes)<br>- <code>asyncio</code>: single-threaded, cooperative concurrency for I/O<br><br>Senior insight: <code>FastAPI</code> with <code>async</code> endpoints handles thousands of concurrent requests on a single thread via the event loop — the GIL is irrelevant there. For CPU-heavy work, use <code>ProcessPoolExecutor</code>.</div>
How does Python's async/await compare to C# and JavaScript?	<div style="text-align:left">All three use the same <code>async</code>/<code>await</code> keywords <code>and</code> event loop concept, but with differences:<br>Python: <code>asyncio</code> event loop, must use <code>async</code>-compatible libraries (<code>httpx</code> <code>not</code> requests, asyncpg <code>not</code> psycopg2)<br>JS: single-threaded by design, all I/O is inherently <code>async</code><br>C#: Task-based, <code>async</code>/<code>await</code> integrates deeply with .NET runtime<br><br>Python gotcha: mixing sync <code>and</code> <code>async</code> is easy to get wrong. Calling a blocking function (time.sleep, requests.get) <code>in</code> an <code>async def</code> blocks the entire event loop. Always use: <code>await asyncio.sleep()</code>, <code>await httpx...</code><br>Use <code>asyncio.run_in_executor()</code> to run sync blocking code from <code>async</code> context.</div>
What is the difference between __repr__ and __str__ in Python classes?	<div style="text-align:left"><code>__str__</code>: human-readable string, called by <code>str()</code> <code>and</code> <code>print()</code><br><code>__repr__</code>: unambiguous developer representation, called by <code>repr()</code> <code>and</code> <code>in</code> the REPL<br><code>class Point:<br>    def __init__(self, x, y): self.x, self.y = x, y<br>    def __str__(self): return f"Point({self.x}, {self.y})"<br>    def __repr__(self): return f"Point(x={self.x!r}, y={self.y!r})"</code><br><br>Rule: <code>__repr__</code> should ideally be valid Python that recreates the object. If only one <code>is</code> defined, <code>__repr__</code> is used as fallback for <code>__str__</code>.<br><br>Senior insight: always implement <code>__repr__</code> <code>in</code> your domain models — it saves hours of debugging.</div>
What is type hinting in Python and why does it matter at the senior level?	<div style="text-align:left">Type hints (PEP 484) are annotations that specify expected types. They don't enforce types at runtime (Python is still dynamic) but:<br>- Enable IDE autocompletion <code>and</code> error detection<br>- Used by <code>FastAPI</code>/<code>pydantic</code> for runtime validation<br>- Improve code readability <code>and</code> maintainability<br>- Allow tools like mypy to do static type checking<br><code>from typing import Optional, List, Dict<br>def get_users(ids: List[int]) -> Optional[Dict[str, str]]:<br>    ...</code><br><br>Senior insight: untyped Python <code>in</code> a large codebase is a maintenance nightmare. Always type hint public APIs <code>and</code> function signatures.</div>
What are Python dataclasses and when to use them vs Pydantic?	<div style="text-align:left"><code>from dataclasses import dataclass<br><br>@dataclass<br>class Point:<br>    x: float<br>    y: float<br>    label: str = "origin"<br><br>p = Point(1.0, 2.0)</code><br>Dataclasses auto-generate <code>__init__</code>, <code>__repr__</code>, <code>__eq__</code>. Use when:<br>- Simple data containers with no validation needed<br>- No JSON serialization needed<br>- Performance matters (faster than <code>pydantic</code>)<br><br>Use Pydantic when:<br>- You need validation, coercion, JSON serialization<br>- Working with <code>FastAPI</code> or external data (user input, API responses)<br><br>Senior insight: dataclasses are stdlib; <code>pydantic</code> is a dependency. Choose based on requirements.</div>
